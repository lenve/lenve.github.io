年后跳槽高峰季，关于简历制作，松哥有话说

刚过完年，又迎来了一波跳槽高峰期。面试的第一关当然就是简历筛选了，一份好的简历能够决定你能够获得多少面试机会，面试机会越多，那你拿一个满意的 offer 的概率才越大。然而实际情况是，很多人对简历不够重视，做出来的简历惨不忍睹。

一般来说，简历做的不好，可能分两种情况：

  • 可能确实是比较水，简历实在写不出个所以然来
  • 技术比较好，但是目空一切，觉得只要技术好，其他都是浮云，这时往往会忽略简历的重要性，导致做出来的简历也乏善可陈。

我们在跳槽找工作的时候，最终能不能拿下 offer ，其实拼的是你的技术实力，这是事物的主要矛盾，但是简历做得好，一方面可以看出你这个人比较认真，对这个岗位比较重视，另一方面可以让你获得足够多的面试机会，有了足够多的面试机会，也就有了足够多的可能性，也就意味着你拿一个令自己的满意的 offer 概率会增大。

今天松哥想聊一点自己的经验，希望能帮助大家把简历的水平再提升一个档次。其实在帮人指导简历这块我的经验还算丰富，有不少成功案例，记得 2018 年有个小伙伴找我看简历，他是当年应届毕业的大专生，后来在深圳拿了一个 10k 的 offer，还有一些 13-14k 的案例，但是不如这个 10k 的典型，我就不一一举例啦。本文文末我还给大家准备了几份简历参考模版，大家在公众号后台回复 简历模板 可以获取下载地址。

简历要认真

这块我刚毕业的时候是有深刻的教训的。2015 年春招的时候，我就属于上边提到的第二种人，那时候心里想着只要咱技术牛逼，简历什么的无所谓。当时在线投的简历由于是在线填写，倒有一些差异，但是线下投递的简历，都是同一个，大概一页纸左右的样子，也没有专门针对某一个公司去做优化，现在想想当时还是太年轻。

一份好的简历，可以有效增加你的面试机会，面试机会多了，才有后面更多的可能性，否则都没人邀请你去面试，那你怎么拿高薪？什么样的简历算是好简历呢？我总结了如下几点：

  1. 个人信息要明确，邮箱、电话、微信不能少，见到过有人投递的简历上面没写电话的，简历倒还不错，可是联系不了，简历只能扔小黑屋了。
  2. 简历一上来要突出优势，例如一些含金量很高的证书、star 数很高的开源项目、访问量很高的博客等等，或者其他一些比较牛 X 的辅助技能，例如我之前见过有个小伙伴英语专八，这在程序员群体中算是比较少见了，像这种比较明显的优势，建议在简历开头就体现出来，先声夺人，不要写在简历结尾。
  3. 简历要真实，会就会，不会就不会。不会的东西你想拿来吹牛，那就下来好好学习一下，千万别写自己不了解的东西。我以前面试过一个，简历上写的熟练掌握 Redis，我问了一个很简单的问题，Redis 的默认端口是什么？结果都答不上来，这一下就让人觉得简历水分很大。
  4. 简历逻辑清晰，很多人写简历写技能点，先来两条 Java 的，再来两条前端的，再来两条数据库的，再来两条前端的，乱七八糟，说明这个人思维很混乱，对程序员来说，思维混乱，肯定写不出逻辑清晰的代码，所以这点也要注意，相同类型的技能点写在一起，当然这属于非常细节的问题了，但是像这些地方其实都体现着程序员的逻辑思维。
  5. 除非对方要求，否则一般建议使用 pdf 格式的简历。word 的话，有时候你用 office 做，对方用 wps 打开，或者你用 wps 做而对方用 word 打开，可能会看到不一样的东西（一般可能只有外包会要求你发 word 简历）。

这是简历的一些要点。简历是一个敲门砖，先赢得面试机会，至于能不能拿下 offer，还得看实力。如果不认真对待简历，那么你能够敲开的门就比别人少，拿高薪的可能性也比别人少。

对照简历准备

简历做好了就完事了吗？肯定不是的。

大家在面试的时候，遇到一些不懂的问题或者没接触过的领域，答不出来这是非常正常的。但是有一种不太正常，那就是简历上写的答不出来，你明明写了你会或者你掌握了某一个技能点，但是对于这个技能点一些非常基础的东西却回答不出来，这是非常要命的，会让人觉得你这个人不老实，简历有水分。

所以，大家在做完简历之后，还需要针对简历上的每一个技能点，去预测关于这个技能点，面试官可能会问什么问题，你的答案又是什么。这即是简历的准备工作，也是一个复习的思路，很多人复习的时候会感到无从下手，其实简历就是一个很好的复习思路，照着简历上写的复习就行了。大部分情况下，面试官都是照着简历去问的，当然也有一些非常二的面试官不看简历天马行空随便问，对于这种情况你就只能自求多福了，你也没法准备，我们能做的，就是尽量在可预测的范围内，提高面试的成功率。

一句话，简历上写的每一条，你都要去预测面试官针对这一条可能问什么问题，而你的答案又是什么？这些我一般建议大家用一个文档记录下来，随时补充总结。针对简历上的每一条都要认真准备，因为在面试过程中，如果简历上写了的，答不出来，这绝对是不可以原谅的。简历之外的，没答出来，我觉得可以理解（因为我也没说我会这个东西呀）。

例如简历上写了：熟练掌握 MySQL、掌握常见的 MySQL 优化技巧。那么你就要复习常见的优化技巧有、索引优化办法、索引为什么失效等。否则当面试官问起这些问题时，你支支吾吾答不出来，就说明你的简历有水分。

及时总结

每一次面试都有有总结文档，面试官问了哪些问题，哪些你答出来了，哪些没答出来，都要总结，没答出来的，请教别人，或者网上查找，要在下一次面试前弄懂，因为同一个问题连续两次面试答不出来或者答错了，也是不可理解不可原谅的。

做好这些，面试应该就是很 easy 的事情了。先把 offer 拿下，到了公司能不能搞定工作，这就看你平时的技术积累了。

老实说，我见过毕业三年的本科只拿到 10k 的，也见过毕业两年的本科拿到 14k（还是跨行），薪水是你综合实力的一个体现，除了日常的技术积累，面试技巧也很重要。

好了，不知道大家有没有 get 到技巧呢。如果小伙伴们也在复习 Spring Boot ，那么不妨看看松哥的视频教程：Spring Boot + Vue 视频教程喜迎大结局，西交大的老师竟然都要来一套！
