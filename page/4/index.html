Git 标签管理

我们可以针对某一次的提交打上一个标签，有点类似于给某次提交取个别名，比如 1.0 版本发布时打个标签叫 v1.0,2.0 版本发布时打个标签叫 v2.0 ，因为每次版本提交的结果都是一连串的哈希码，不容易记忆，打上 v1.0,v2.0 这些具有某种含义的标签后，可以方便我们进行版本管理。
